beauty_health_demo v23 使い方メモ

【今回の追加機能】
1. 成分名の混在判定を強化
   - 日本語名
   - INCI名
   - 英語名
   - aliases（別名・略称・OCRで出やすい表記ゆれ）
   を同じ成分として判定できるようにしました。

2. 推奨成分UIを改善
   - 推奨成分を機能グループごとに表示
   - グループ内は点数が高い順に左から表示
   - グループ単位で一括選択 / 解除が可能

【重要】
HTMLを開いたあと、成分辞書を更新したいときは ingredients_template.xlsx を編集して
管理者画面から取り込んでください。

--------------------------------------------------
ingredients_template.xlsx の新しい列
--------------------------------------------------

domain
  対象ジャンル
  - cosmetics
  - healthfood
  - both
  を入力します。

name
  内部の基準名です。
  比較・理想設計・製品との一致判定の基準になります。
  日本語の代表表記を入れるのがおすすめです。

inci
  INCI名です。
  例: Sodium Hyaluronate / Niacinamide / Glycerin
  製品全成分がINCI名で記載されている場合の一致判定に使います。

english
  英語の一般名です。
  INCIと同じ場合もありますが、別管理にしています。

name_en
  英語表示時の画面用名称です。
  未入力なら name を表示します。

name_zh
  中国語表示時の画面用名称です。
  未入力なら name を表示します。

aliases
  別名・略称・表記ゆれを「、」区切りで入れます。
  例:
  ブチレングリコール、Butylene Glycol、1,3-Butylene Glycol
  ここに入れた名称は、全成分解析や比較で同一成分として扱います。

tags
  機能タグです。「、」区切り。
  推奨欄・成分一覧・カード表示に使います。

role
  主役 / 補助 / 基盤 など。

short
  短い説明文。カード概要で使います。

detail
  詳しい説明文。詳細表示で使います。

merits
  メリットを「、」区切りで入れます。

cautions
  注意点を「、」区切りで入れます。

--------------------------------------------------
数値列の意味
--------------------------------------------------

化粧品向け
- moisture     保湿
- barrier      バリア
- brightening  透明感
- firmness     ハリ
- soothing     整肌

健康食品向け
- beauty       美容
- rest         休息
- gut          腸活
- vitality     活力
- circulation  巡り

数値は絶対値ではなく、相対的な寄与スコアです。
0〜40程度で運用すると扱いやすいです。
数字が高いほど、その機能グループで上位に出やすくなります。

--------------------------------------------------
推奨成分がどう並ぶか
--------------------------------------------------

1. 悩み・カテゴリ・使用感などから提案点を計算
2. score が高い成分を候補に残す
3. 各成分について、もっとも高い機能スコアのグループへ分類
   例:
   - セラミド → バリア
   - ヒアルロン酸Na → 保湿
   - GABA → 休息
4. グループ内では提案点が高い順に左から表示

そのため、成分辞書を強化しても、機能グループごとに整理されて見やすいまま使えます。

--------------------------------------------------
更新のおすすめ手順
--------------------------------------------------

1. まず name を代表表記で決める
2. inci / english / aliases を埋める
3. tags / short / detail を入れる
4. 数値列を入れる
5. 管理者画面から取り込む
6. 理想設計と比較で判定できるか確認する

--------------------------------------------------
注意
--------------------------------------------------

- aliases に入っていない表記は、同一成分として判定されないことがあります
- OCRは完全ではないため、表記ゆれの多い成分は aliases を厚くするのがおすすめです
- HTML単体版なので、Excel取込とOCRライブラリの読込にはネット接続が必要です
